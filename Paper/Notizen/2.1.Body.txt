2.0	Definitionen der Datenstrukturen
	2.1 	ControlSet 

Ein Control Set ist eine Menge von allen möglichen Bewegungen, die Minimalisiert und Zerteilt wurden.
Dadurch sind die Bewegungen nicht subsituierbar durch andere Bewegunge im Set.

Das Control Set wird von suchbasierten Algorithmen genutzt.
Ein Grund für die Nutzung des Control Sets wäre, dass an Zeit und Rechenaufwand für die lokale Plannung  gesparrt wird, weil das erstellen eines Control Sets einmalig ist und das Set wieder verwendbar ist.
Weitere Nutzen des Control Sets werden im Paper #VERWEIS_HIER erwähnt.

Im Code wird das Control Set als eine Map von Positionen und Bewegungen implementiert. 

Bei der Erzeugung eines Control Sets wird auf den Bewegungserzeuger und einem Intervall, den man selbst bestimmt, geachtet. 

Bei der Nutzung des Control Sets ist es Vorteilhaft, dass die Bewegungen in dieser Art Abgespeichert werden, weil man sich schnell und gezielt Bewegungen in Abhängigkeit der Position aus dem Control Set aufrufen kann.

Auch ist das Control Set abhängig von den Bewegungserzeugern, was dem Nutzer die Freiheit gibt, den Code anzupassen.

 Zum Beispiel kann der Benutzer zwei Control Set generieren lassen mit unterschiedlichen Bewegungserzeugern und den selben Intervall und diese miteinander vergleichen.
